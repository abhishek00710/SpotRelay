# Contributing

Thanks for contributing to SpotRelay.

## Development Principles

- Keep the UX fast, calm, and obvious.
- Prefer small, reviewable pull requests.
- Preserve the map-first flow and avoid unnecessary screen complexity.
- Document product or architectural tradeoffs in the PR description.

## Setup

1. Open `SpotRelay.xcodeproj`.
2. Build and run the `SpotRelay` scheme.
3. Use the test targets when changing state logic or user flows.

## Pull Requests

- Describe the user-facing change clearly.
- Include screenshots or short recordings for UI changes.
- Call out any new warnings, limitations, or follow-up work.
- Add or update tests when changing store logic or state transitions.

## Code Style

- Follow Swift and SwiftUI defaults already used in the repo.
- Keep files focused on one feature or responsibility.
- Prefer readable names over clever abstractions.

## Issue Reporting

When filing a bug, include:

- What you expected
- What happened instead
- Device or simulator details
- Screenshots or console logs if relevant
